# 인터넷 조사 — 키워드 협업 루프

유저와 키워드를 협의하며 검색 정밀도를 점진적으로 높이는 조사 프로세스.

## 왜 이 프로세스가 필요한가

- AI가 혼자 키워드를 추측하면 첫 키워드 품질에 결과가 좌우됨
- 유저는 결과를 보면서 방향을 잡는 게 자연스러움
- 1차 결과를 공유하고 유저가 수정하는 루프로 정밀도를 올린다

## Step 0: 스킵 판단

조사 요청이 들어오면 먼저 판단한다:

- **공식 문서 직행** (예: "React useState 사용법", "npm 명령어", 특정 API 스펙)
  → Context7로 바로 답하고 이 스킬을 종료한다
- **탐색이 필요한 주제** (예: "요즘 카페 게임 트렌드", "이런 UX 패턴 사례", "경쟁사 분석")
  → Step 1로 진행

**판단 기준:** 답이 하나의 공식 소스에 있으면 스킵. 여러 소스를 비교/종합해야 하면 진행.

## Step 1: 1차 검색 (AI 자율)

유저 개입 없이 자율적으로 탐색한다.

1. 주제에서 키워드 5~7개를 도출한다
   - 한국어 / 영어 각각
   - 동의어, 업계 용어 변형 포함
2. WebSearch로 검색한다
3. 신뢰도 높은 결과를 WebFetch로 읽는다
4. Step 2로 넘어간다

## Step 2: 중간 공유

아래 포맷으로 결과를 유저에게 보여준다:

```
## N차 검색 결과

**사용한 키워드:** "keyword1", "keyword2", ...

| # | 출처 | URL | 핵심 내용 (1줄) | 신뢰도 |
|---|------|-----|----------------|--------|
| 1 | example.com | https://example.com/article | 설명... | ★★★★ |
| 2 | reddit.com/r/... | https://reddit.com/r/.../... | 설명... | ★★★ |

**아직 못 찾은 것:** ~에 대한 정보는 부족
**다음 라운드 제안 키워드:** "○○", "△△"

→ 방향 수정해줘, 또는 "됐어" 하면 종합 정리할게
```

**필수 포함 항목:**
- 사용한 키워드 전체 목록
- 각 결과의 URL (유저가 직접 확인 가능하도록)
- 빈 곳: 검색 결과 3건 미만인 소주제는 "부족"으로 표시
- 다음 라운드에 시도할 키워드 제안

## Step 3: N차 검색 (유저 가이드)

유저 피드백을 반영해서 검색한다:

- "3번 방향 더 파봐" → 해당 소주제 키워드 확장
- "키워드에 ○○ 추가" → 유저 키워드를 시드로 변형 생성
- "이쪽은 됐고 저쪽 더" → 방향 전환

검색 후 Step 2로 돌아간다 (반복).
유저가 "됐어"를 말하면 Step 2로 돌아가지 않고 바로 Step 4로 진행한다.

**라운드 운영 방침:**
- 라운드 수 고정 없음 — 유저가 "됐어" 할 때까지 반복
- 3라운드 초과 시 "이 정도면 충분한지" 한 번 확인
- 유저가 계속하자고 하면 계속 진행

## Step 4: 종합 정리

유저가 "됐어"로 조사를 종료하면 최종 종합한다:

```
## 조사 종합: [주제]

### 핵심 인사이트
1. ...
2. ...
3. ...

### 주제별 정리
#### [소주제 1]
- ...

#### [소주제 2]
- ...

### 출처 목록
| # | 출처 | URL | 신뢰도 |
|---|------|-----|--------|
| 1 | ... | ... | ★★★★ |
```

## 신뢰도 기준

| 등급 | 기준 |
|------|------|
| ★★★★★ | 공식 문서, 1차 출처 (논문, 공식 블로그) |
| ★★★★ | 업계 미디어 (GameIndustry.biz, GDC 발표 등) |
| ★★★ | 커뮤니티 합의 (Reddit 상위글, Stack Overflow) |
| ★★ | 개인 블로그, 의견성 글 |
| ★ | 출처 불명, 오래된 정보 |

## 제약 사항

- **로그인 필요 사이트**: 접근 불가 — 발견 시 유저에게 알림
- **유료 콘텐츠**: 초록만 확인 가능
- **URL 필수**: 모든 결과에 URL 포함
- **도구 실패 시**: 범위 축소 → 우회 → 유저와 대안 논의 (CLAUDE.md 복구 순서)
- **`/browse` 미사용**: 이 스킬은 WebSearch/WebFetch를 직접 호출함. gstack `/browse`는 QA/탐색용으로 별도

## 하지 말 것

- 1차 검색 결과를 유저에게 보여주지 않고 바로 종합하지 않는다
- 유저 피드백 없이 혼자 2차 검색으로 넘어가지 않는다
- 중간 공유에서 URL을 생략하지 않는다
- 검색 결과가 없을 때 추측으로 채우지 않는다 — "못 찾았다"고 솔직히 표시
